# 📖 Ультра-простой гайд по автопостингу для новичков

> **Кейс от [@socialmediamisha](https://github.com/socialmediamisha)**

Этот гайд написан специально для тех, кто никогда не работал с кодом, серверами или консолью. Здесь всё расписано «на пальцах»: куда нажимать, что копировать и как запустить автоматический постинг за 15 минут.

---

## 🧭 Карта настройки: 5 простых этапов

1. **Скачивание программы** к себе на компьютер.
2. **Установка Node.js** (это база, на которой работает наш скрипт).
3. **Получение ключей от Threads** (чтобы скрипт мог публиковать посты в ваш аккаунт).
4. **Получение ключа от ИИ Gemini** (чтобы посты писались красивым языком).
5. **Запуск в облаке GitHub бесплатно** (чтобы скрипт работал сам 24/7).

---

## 🛠️ Шаг 1. Скачиваем программу и ставим Node.js

1. На странице этого проекта на GitHub нажмите зеленую кнопку **Code** вверху справа и выберите **Download ZIP** (Скачать ZIP-архив).
2. Распакуйте скачанный архив в любую папку на вашем компьютере (например, на Рабочий стол).
3. Установите программу **Node.js**:
   - Перейдите на сайт [nodejs.org](https://nodejs.org/).
   - Нажмите на большую кнопку с надписью **LTS** (это самая стабильная версия).
   - Скачайте установщик, запустите его и просто нажимайте *Далее (Next) -> Установить (Install)* до самого конца. Никакие галочки менять не нужно.

---

## 🔑 Шаг 2. Регистрируем приложение в Threads

Чтобы программа могла отправлять посты в ваш Threads, нужно получить разрешение от компании Meta.

1. Перейдите на сайт для разработчиков: [developers.facebook.com](https://developers.facebook.com/).
2. Зарегистрируйтесь как разработчик (просто подтвердите свою почту и нажмите «Далее»).
3. Создайте приложение:
   - Нажмите **Создать приложение (Create App)**.
   - Выберите вариант **Другое (Other)** -> нажмите **Далее**.
   - На следующем шаге выберите **Threads** -> нажмите **Далее**.
   - Введите любое название приложения (например, `MyAutoPoster`) и укажите вашу почту. Нажмите **Создать приложение**.
4. В левом меню нажмите **Настройки (Settings)** -> **Базовые (Basic)**.
   - Здесь вы увидите два поля: **Идентификатор приложения (App ID)** и **Секрет приложения (App Secret)** (нажмите «Показать» и введите пароль от Facebook, чтобы увидеть его).
   - Скопируйте оба этих значения в блокнот. Они нам сейчас понадобятся.
5. Нажмите на вкладку **Threads** в левом меню:
   - Вставьте адрес `https://localhost:3000/` в поля redirect-урлов (это обязательно для авторизации).
   - Прокрутите страницу ниже до раздела **Тестировщики приложений (App Testers)**. Нажмите «Добавить тестировщика» и введите ваш никнейм в Threads.
6. Откройте приложение Threads на вашем телефоне (или сайт threads.net):
   - Зайдите в *Настройки -> Версия для разработчиков -> Тестировщики приложений*.
   - Вы увидите приглашение от вашего приложения. Нажмите **Принять (Accept)**.

## ⚡ Шаг 3. Получаем токен доступа (Ключ к Threads)

Официальный токен Meta работает всего 1 час. Но наша программа умеет превращать его в **60-дневный токен**, который будет работать два месяца!

1. Зайдите в папку с распакованным проектом.
2. Найдите файл с именем `.env.example`.
3. Переименуйте его. Новое имя должно быть ровно `.env` (без `.example` на конце).
4. Теперь откройте командную строку (консоль) вашего компьютера:
   * **На Windows**: Откройте папку с файлами проекта, кликните по пустому месту в адресной строке проводника вверху, сотрите всё, напишите `cmd` и нажмите Enter.
   * **На Mac**: Нажмите правой кнопкой мыши по папке с проектом -> *Службы -> Новый терминал в папке*.
5. Введите команду и нажмите Enter:
   ```bash
   npm run setup
   ```
6. Терминал попросит вас ввести **Threads App ID** — вставьте его (нажмите правой кнопкой мыши для вставки в Windows) и нажмите Enter.
7. Затем вставьте **Threads App Secret** и нажмите Enter.
8. Скрипт покажет вам длинную интернет-ссылку. **Выделите её, скопируйте и откройте в браузере**.
9. На экране появится запрос на авторизацию Threads. Нажмите **Разрешить (Allow)**.
10. После этого браузер попытается загрузить страницу `https://localhost:3000/` и выдаст ошибку вроде «Не удается получить доступ к сайту». **ЭТО НОРМАЛЬНО! Не пугайтесь.**
11. Посмотрите на адресную строку браузера. Она изменилась и теперь выглядит примерно так:
    `https://localhost:3000/?code=AQDb8...#_=_`
12. **Скопируйте эту ссылку целиком** из адресной строки, вернитесь в окно консоли, вставьте её туда и нажмите Enter.
13. Скрипт сам сделает всю магию! На экране появятся строчки:
    - `THREADS_ACCESS_TOKEN=...`
    - `THREADS_USER_ID=...`
14. Откройте ваш файл `.env` через обычный Блокнот и вставьте туда эти два значения вместо текста `your_long_lived_access_token` и `your_threads_user_id`.

## 🤖 Шаг 4. Получаем ключ ИИ (Gemini API Key)

Чтобы посты писались бесплатно через лучшую нейросеть от Google:

1. Перейдите на сайт: [aistudio.google.com](https://aistudio.google.com/).
2. Войдите под своим Google-аккаунтом.
3. Нажмите синюю кнопку **Get API key** (Получить API ключ) слева вверху.
4. Нажмите **Create API key** -> Выберите любой проект -> Нажмите **Create API key in existing project**.
5. Скопируйте полученный ключ (длинная строка букв и цифр).
6. Откройте файл `.env` в Блокноте и вставьте этот ключ в строчку `GEMINI_API_KEY=ваш_ключ`.

## ☁️ Шаг 5. Запуск в облаке GitHub бесплатно (Автопилот 24/7)

Теперь сделаем так, чтобы всё работало само на серверах GitHub без вашего участия.

### 1. Создаем репозиторий
1. Зарегистрируйтесь на сайте [github.com](https://github.com/).
2. Нажмите **New** (Создать новый репозиторий).
3. Введите любое имя (например, `my-threads-poster`).
4. **Важно:** Выберите пункт **Private** (Приватный), чтобы никто не увидел ваши секретные ключи!
5. Нажмите **Create repository**.
6. GitHub покажет вам список команд. Если вы не хотите работать с Git через консоль, просто перетащите все файлы из распакованной папки вашего компьютера прямо в окно браузера на сайте GitHub, чтобы загрузить их, и нажмите кнопку **Commit changes** внизу.
   *(Внимание: файл `.env` перетаскивать НЕ надо! Он должен оставаться только у вас на компьютере).*

### 2. Добавляем секреты
Чтобы GitHub Actions знал ваши пароли, но они были в безопасности:
1. В вашем репозитории на сайте GitHub нажмите вкладку **Settings** (Настройки) сверху.
2. В меню слева найдите **Secrets and variables** -> кликните на **Actions**.
3. Нажмите зеленую кнопку **New repository secret** (Создать секрет).
4. Создайте три секрета по очереди:
   * Название: `THREADS_ACCESS_TOKEN` | Значение: скопируйте токен из вашего `.env`.
   * Название: `THREADS_USER_ID` | Значение: скопируйте ID пользователя из вашего `.env`.
   * Название: `GEMINI_API_KEY` | Значение: скопируйте ключ Gemini из вашего `.env`.

### 3. Даем права на запись
Скрипт должен уметь обновлять историю постов и удалять опубликованные черновики. Для этого:
1. В левом меню настроек GitHub (Settings) выберите раздел **Actions** -> **General**.
2. Прокрутите страницу в самый низ до заголовка **Workflow permissions**.
3. Поставьте точку напротив пункта **Read and write permissions** (Разрешить чтение и запись).
4. Нажмите кнопку **Save** (Сохранить).

### 4. Настраиваем время публикации (cron-job.org)
Осталось настроить будильник, который будет пинать GitHub запустить скрипт.
1. Зарегистрируйтесь на бесплатном сайте [cron-job.org](https://cron-job.org/).
2. Нажмите кнопку **Create Cronjob** (Создать задачу):
   * **Title**: `Threads Post`
   * **URL**: `https://api.github.com/repos/ВАШ_НИК_НА_GITHUB/ИМЯ_ВАШЕГО_РЕПОЗИТОРИЯ/dispatches`
     *(Замените ВАШ_НИК_НА_GITHUB и ИМЯ_ВАШЕГО_РЕПОЗИТОРИЯ на свои данные)*
   * **Execution schedule**: Выберите нужное вам время (например, каждый день в 10:00 утра).
   * **Request method**: Выберите `POST`.
   * В разделе **Request headers** (Заголовки запроса) нажмите «Add header» и добавьте 4 строки:
     1. Ключ: `Accept` | Значение: `application/vnd.github+json`
     2. Ключ: `Authorization` | Значение: `Bearer ВАШ_ТОКЕН_GITHUB`
        *(Как получить токен GitHub: зайдите в Settings вашего профиля GitHub -> Developer settings -> Personal access tokens -> Tokens (classic) -> Generate new token (classic). Поставьте галочку на пункт `repo` и нажмите сгенерировать. Скопируйте его).*
     3. Ключ: `Content-Type` | Значение: `application/json`
     4. Ключ: `User-Agent` | Значение: `cron-job`
   * В поле **Request body** (Тело запроса) вставьте вот этот текст:
     ```json
     {
       "event_type": "daily_post"
     }
     ```
   * Нажмите кнопку **Create** внизу.

🎉 **Всё готово!** Теперь каждый день в выбранное вами время сервис cron-job будет посылать сигнал на GitHub, и ваш аккаунт Threads будет автоматически публиковать свежий, красиво написанный ИИ пост.

## 🎨 Как настроить тематику и источники под себя (4 режима работы)

Вы можете настроить бота так, как вам удобно. Все конфигурации делаются в самом начале файла `publish.js` в блоке `=== НАСТРОЙКА ДЛЯ ПОЛЬЗОВАТЕЛЯ (CUSTOMIZATION) ===`. Откройте этот файл в Блокноте и настройте один из 4 режимов:

---

### Режим 1. Кросс-постинг из вашего Telegram-канала или блога
Бот будет автоматически брать ваши посты из Telegram, YouTube или блога, переписывать их с помощью ИИ под формат Threads и публиковать.

*   **Настройка `RSS_FEEDS`**:
    ```javascript
    const RSS_FEEDS = [
      'https://rsshub.app/telegram/channel/ИМЯ_ВАШЕГО_ТГ_КАНАЛА' // Для Telegram
      // 'https://username.substack.com/feed'                      // Для Substack блога
      // 'https://medium.com/feed/@username'                      // Для Medium блога
    ];
    ```
    *ИИ Gemini автоматически уберет лишние ссылки, адаптирует текст под формат Threads и сократит его до лимита в 500 символов.*

---

### Режим 2. Автопилот на новостях (Tech, Бизнес, Спорт и др.)
Бот будет находить свежие статьи на выбранных вами новостных ресурсах, переводить их на русский язык и писать экспертное саммари от вашего имени.

*   **Настройка `RSS_FEEDS`**:
    Укажите ссылки на любые RSS-ленты новостей (на английском или русском):
    ```javascript
    const RSS_FEEDS = [
      'https://techcrunch.com/feed/', // Англоязычный TechCrunch
      'https://vc.ru/rss',           // Русскоязычные новости бизнеса и стартапов
      'https://habr.com/ru/rss/all/all/' // Новости Хабра
    ];
    ```

---

### Режим 3. Микс (Свои посты + Новости интернета)
Бот будет случайно выбирать между вашими публикациями из Telegram/блога и мировыми техно-новостями.

*   **Настройка `RSS_FEEDS`**:
    Впишите в массив и свои соцсети, и новостные сайты через запятую:
    ```javascript
    const RSS_FEEDS = [
      'https://rsshub.app/telegram/channel/my_channel_name',
      'https://vc.ru/rss'
    ];
    ```

---

### Режим 4. Полностью автономный ИИ-блогер (Без сайтов)
Бот не будет ничего парсить. Он будет сам ежедневно генерировать интересные посты на тему, которую вы ему зададите.

*   **Настройка `RSS_FEEDS`**: Оставьте массив абсолютно пустым:
    ```javascript
    const RSS_FEEDS = [];
    ```
*   **Настройка `FALLBACK_AI_PROMPT`**: Напишите тему, на которую бот должен рассуждать:
    ```javascript
    const FALLBACK_AI_PROMPT = `
    Сгенерируй короткую интересную мысль, наблюдение или тренд из сферы IT, SMM или искусственного интеллекта.
    `;
    ```
    *(Вы можете вписать любую тему: рецепты, советы по фитнесу, инвестиции, цитаты, факты об истории).*

---

### 📝 Внеочередные черновики (`thoughts.txt`)
В любом из режимов, если вы запишете мысль в файл `thoughts.txt` на вашем GitHub, бот при следующем запуске проигнорирует все новостные ленты, возьмет вашу запись, красиво причешет её через ИИ и опубликует вне очереди.

После изменения настроек в `publish.js` просто сделайте коммит и отправьте файл на GitHub.
