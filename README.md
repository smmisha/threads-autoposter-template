# 🚀 Автопостинг в Meta Threads с ИИ Gemini и RSS-новостями

> **Готовый кейс от [@socialmediamisha](https://github.com/socialmediamisha)**
> 
> 📖 **[УЛЬТРА-ПРОСТОЙ ПОШАГОВЫЙ ГАЙД ДЛЯ ТЕХ, КТО СОВСЕМ НЕ УМЕЕТ В КОД](GUIDE.md)**

Этот проект позволяет полностью автоматизировать ведение вашего аккаунта в **Threads**. Скрипт сам забирает свежие новости из RSS-ленты, отправляет их ИИ (Google Gemini) для перевода, фильтрации и гуманизации (написание живым, человеческим языком от первого лица), а затем публикует готовый пост в ваш аккаунт Threads через официальный API.

Вся система работает **абсолютно бесплатно** и круглосуточно на базе GitHub Actions и внешнего триггера. Вам не нужно держать включенным компьютер или арендовать сервер!

---

## ✨ Основные возможности

1. **Полная автоматизация**: Посты выходят сами по расписанию.
2. **Умный ИИ-копирайтер**: Google Gemini 3.5/2.5 Flash переписывает сухие новости в живые посты с личным мнением, избегая шаблонных фраз искусственного интеллекта ("важно отметить", "в современном мире" и т.д.).
3. **Безопасность и фильтрация**: Встроенная система фильтрации исключает из ленты темы политики, религии, войн и оборонных технологий.
4. **Резервные черновики**: Возможность закинуть свои идеи/заготовки в файл `thoughts.txt` — ИИ сам превратит их в красивые посты и опубликует в первую очередь.
5. **Бесплатный деплой**: Запуск через GitHub Actions и планировщик без затрат на хостинг.
6. **Надежность**: Автоматические повторные попытки (retry) с задержкой при временных сбоях API Meta Threads или Google Gemini.

## 🛠️ Быстрый старт (Локальная настройка)

### Шаг 1. Клонирование и подготовка
1. Склонируйте этот репозиторий:
   ```bash
   git clone https://github.com/ВАШ_НИК/threads-autoposter.git
   cd threads-autoposter
   ```
2. Скопируйте шаблон конфигурации в рабочий файл `.env`:
   ```bash
   cp .env.example .env
   ```

*(Примечание: В проекте нет внешних npm-зависимостей, всё работает на нативных возможностях Node.js 18+, поэтому выполнять `npm install` не требуется!)*

---

### Шаг 2. Настройка приложения Meta (Threads API)
Для отправки постов нужен официальный доступ к Threads API:
1. Перейдите на портал [Meta for Developers](https://developers.facebook.com/) и зарегистрируйте аккаунт разработчика.
2. Создайте новое приложение. При выборе типа приложения укажите **Другое (Other)**, а на следующем шаге выберите **Threads**.
3. В панели управления приложением перейдите в настройки Threads:
   - Добавьте ваш аккаунт Threads в качестве **Тестировщика Threads (Threads Tester)** в настройках приложения.
   - Примите инвайт в настройках вашего аккаунта Threads (Настройки -> Версия для разработчиков -> Тестировщики приложений).
   - Скопируйте **Threads App ID** (Идентификатор приложения) и **Threads App Secret** (Секрет приложения).
   - Задайте адрес перенаправления (Redirect URI) в настройках Threads API внутри Meta: `https://localhost:3000/`.

---

### Шаг 3. Интерактивная генерация долговечного токена
Meta выдает короткие токены доступа, которые сгорают через час. Нам нужен долговечный токен на 60 дней. Для этого в проекте есть удобный скрипт-помощник:

1. Запустите скрипт авторизации:
   ```bash
   npm run setup
   ```
2. Скрипт спросит у вас:
   - **Threads App ID**
   - **Threads App Secret**
3. После этого скрипт сгенерирует и выведет специальную ссылку. Скопируйте её и откройте в браузере.
4. Авторизуйте ваше приложение. Браузер перенаправит вас на несуществующую страницу (это нормально).
5. **Скопируйте всю адресную строку** (она будет выглядеть как `https://localhost:3000/?code=AQ...#_=_`) и вставьте обратно в терминал скрипта.
6. Скрипт автоматически:
   - Обменяет временный код авторизации на короткий токен.
   - Обменяет короткий токен на **долговечный (60 дней)**.
   - Получит ваш **Threads User ID**.
   - Покажет готовые значения для файла `.env` и GitHub Secrets!

Запишите полученные значения `THREADS_ACCESS_TOKEN` и `THREADS_USER_ID` в ваш файл `.env`.

---

### Шаг 4. Получение ключа Google Gemini API
1. Перейдите в [Google AI Studio](https://aistudio.google.com/).
2. Создайте бесплатный API-ключ (**Get API key**).
3. Добавьте его в файл `.env` как `GEMINI_API_KEY`.

## 🚀 Деплой в облако (GitHub Actions + Автоматизация)

Чтобы скрипт работал сам каждый день, мы настроим его выполнение на серверах GitHub.

### Шаг 1. Создание приватного репозитория
1. Создайте **приватный** (Private) репозиторий на вашем аккаунте GitHub.
2. Залейте файлы проекта в ваш новый репозиторий:
   ```bash
   git init
   git remote add origin https://github.com/ВАШ_НИК/threads-autoposter.git
   git branch -M main
   git add .
   git commit -m "Initial commit"
   git push -u origin main
   ```

---

### Шаг 2. Настройка секретов репозитория
Ни в коем случае не выкладывайте ваши токены и ключи в открытый доступ! Для этого на GitHub есть раздел секретов:
1. Откройте ваш репозиторий на GitHub и перейдите в **Settings** -> **Secrets and variables** -> **Actions**.
2. Создайте три секрета (**New repository secret**):
   - `THREADS_ACCESS_TOKEN` — ваш долговечный токен из `setup.js`.
   - `THREADS_USER_ID` — ваш цифровой ID пользователя Threads.
   - `GEMINI_API_KEY` — API-ключ от Google AI Studio.

---

### Шаг 3. Настройка разрешений для GitHub Actions
Скрипт обновляет файл `thoughts.txt` (удаляет опубликованный черновик), поэтому GitHub-боту нужны права на запись изменений в ваш репозиторий:
1. В настройках репозитория перейдите в **Settings** -> **Actions** -> **General**.
2. Прокрутите страницу вниз до раздела **Workflow permissions**.
3. Выберите опцию **Read and write permissions** и сохраните изменения.

---

### Шаг 4. Настройка стабильного триггера (cron-job.org)
Стандартный планировщик задач в GitHub Actions (cron) для бесплатных аккаунтов может срабатывать со значительным опозданием или пропускать запуски. Для точного постинга минута в минуту мы используем внешний бесплатный сервис:

1. Создайте личный токен доступа GitHub (Personal Access Token - PAT):
   - Перейдите в **Settings** (настройки вашего профиля GitHub) -> **Developer settings** -> **Personal access tokens** -> **Tokens (classic)**.
   - Нажмите **Generate new token (classic)**.
   - В поле **Note** укажите, например, `threads-autoposter-trigger`.
   - Выберите срок действия (например, 90 дней или без срока).
   - Отметьте галочкой область **repo** (полный доступ к репозиториям).
   - Скопируйте созданный токен (он будет показан один раз).
2. Зарегистрируйтесь на сайте [cron-job.org](https://cron-job.org/).
3. Создайте новую задачу (**Create Cronjob**):
   - **Title**: `Threads Autoposter`
   - **URL**: `https://api.github.com/repos/ВАШ_НИК/НАЗВАНИЕ_РЕПОЗИТОРИЯ/dispatches`
   - **Execution schedule**: Выберите нужное вам время (например, раз в сутки в 12:00).
   - **Request method**: `POST`
   - **Request headers**: Добавьте 4 обязательных заголовка:
     1. `Accept`: `application/vnd.github+json`
     2. `Authorization`: `Bearer ВАШ_GITHUB_PERSONAL_ACCESS_TOKEN` (замените на ваш токен)
     3. `Content-Type`: `application/json`
     4. `User-Agent`: `cron-job`
   - **Request body**: Вставьте JSON-строку:
     ```json
     {
       "event_type": "daily_post"
     }
     ```
   - Нажмите **Create**.

Теперь сервис cron-job.org будет посылать сигнал на GitHub точно по расписанию, мгновенно запуская процесс публикации.

## 📝 Использование черновиков (`thoughts.txt`)

Если вы хотите опубликовать конкретную мысль вместо случайной новости из интернета:
1. Откройте файл `thoughts.txt`.
2. Запишите в него вашу мысль. Если мыслей несколько, разделите их строкой из трех дефисов `---`. Например:
   ```text
   Сегодня думаю о том, как искусственный интеллект меняет подход к созданию контента. Блогинг уже никогда не будет прежним.
   ---
   Запустил автопостинг в Threads на чистом Node.js. Скрипт работает без серверов на GitHub Actions!
   ```
3. Сделайте коммит и отправьте изменения на GitHub.
4. При следующем запуске скрипт возьмет верхнюю заготовку из файла `thoughts.txt`, пропустит её через Gemini (чтобы сделать текст более живым и убрать AI-клише), опубликует в Threads, а затем удалит её из файла `thoughts.txt`, автоматически сделав коммит в ваш репозиторий.

## 🎨 Настройка под свою тематику (Кастомизация)

По умолчанию скрипт берет IT-новости с сайта TechCrunch. Но вы можете легко изменить его под любую тематику:

1. **Изменение RSS-ленты новостей**:
   * Откройте файл `publish.js` и найдите строчку (примерно 153-я строка):
     ```javascript
     const RSS_FEED_URL = 'https://techcrunch.com/feed/';
     ```
   * Замените адрес на любую другую RSS-ленту. Например, на `https://vc.ru/rss` (бизнес/стартапы) или `https://habr.com/ru/rss/all/all/` (IT на русском).

2. **Изменение темы резервной генерации**:
   * Если новостей нет или сайт недоступен, ИИ генерирует пост сам. Изменить тему можно в файле `publish.js` (примерно 281-я строка):
     ```javascript
     prompt = `
     Сгенерируй короткую интересную мысль, наблюдение или тренд из сферы IT, SMM или искусственного интеллекта.
     `;
     ```
   * Замените текст на вашу тематику (например, советы по фитнесу, инвестициям, кулинарии и т.д.).
